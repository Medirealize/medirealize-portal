# 2026-04-14 の実践Tips

## 今日の要点
- リファクタリングは「分離・型付け・削除」を同じタイミングで行うと、整理後に「なぜここに置いたか」が明確になる
- AI 生成の問いは「視点の被り」に対して明示的に制約をかけないと、同じ切り口が並ぶことがある
- fallback 処理の空文字列 split バグは、発見時点で即修正しないと連鎖して気づきにくくなる

## そのまま使える手順
1. ロジック分離時は型定義を先に書いてから実装を移すと、型の曖昧さが移動しない
2. `as any` を置き換えるときは API のレスポンス型を公式ドキュメントか SDK の型定義から引っ張る
3. AI 生成プロンプトに「それぞれ異なる視点から〇問提示して」を追加するだけで多様性が向上する
4. モバイル向けの問い文は「20〜35文字程度」を目安にすると2行以内に収まりやすい
5. 不要ファイルの削除は依存関係を `grep` で確認してから行う

## 一言メモ
コードを動かすのは1回だが、読むのは何度も繰り返す。リファクタリングは次の機能追加を速くするための先行投資だ。
