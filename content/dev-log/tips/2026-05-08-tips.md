# 2026-05-08 の実践Tips

## 今日の要点
- フォーム系UIの「使いやすさ」は、プログレス・最短経路・入力補助・プライバシー・アクセシビリティ・エラー回復の6点で診断できる
- 変更ファイルが1本でも、観点を揃えれば体験は大きく変わる
- エラー時の再送導線は「最後に使ったモード」を記憶することで、ユーザーの作業を巻き戻さずに済む

## そのまま使える手順
1. 進行バーは `(currentStep / totalSteps) * 100` の比率計算で実装し、ステップ数を固定しておく
2. 送信モードを `type SubmitMode = "normal" | "quick"` で型定義し、同じ処理関数に引数として渡す
3. プライバシー説明は `useState<boolean>(false)` で展開フラグを持ち、短文と詳細をトグル切替にする
4. フォントサイズ切替は `mainTextClass / subTextClass / noteTextClass` のようにクラスを変数に束ねて JSX で使い回す
5. エラー時は `lastSubmitMode` を state で保持し、再送ボタンから `retryLastSubmit()` を呼ぶ

## 一言メモ
「動く」と「使いたい」の差は機能の数ではなく、迷わせない・見通しを渡す・信頼を途切れさせない設計の積み重ねにある。
